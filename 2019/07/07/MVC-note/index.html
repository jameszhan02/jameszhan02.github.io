MVC Structure:

mvc stand for Model, View and controller.

See in the pic have three different folder call Models, Views, and Conrollers.

As the end user, they only see the view.

View :

ASP.NET is dynamic website. Whcih means, Most of views(HTML) Generate from the databse.

View is the the final HTML will be display to the end user. Consider View is the folder to hold HTML Layout.

All files are cshtml inside.

Controller :

controller is the component between model and view. all of action from end user will be send controller, and controller get info/data from the model and to build the view for the final display.

Notice Controller should share the same name with the folder in the View.

For example, if there is a folder call Home inside of the view folder. then in controller suppose to have one call HomeController to handle Home (View).

This is the default view controller method looks like. it’s just simpleily return the view page to the end user when the controller be called(request)

Model :

Models are the class to hold the data. For example, When we transfer JSON File into the C#, we need to create a new “Data type” to hold those data in the .json and that is the model.

This is an example model i did for holding product. Notice keywords in [] belong to SQL. is define SQL type inside C#.

summary :

  • Model
    • Data related logic
    • Interations with database
    • communcates with controller
    • can sometimes update the view
  • View
    • is what end user sees
    • Usually consists of HTML/CSS
    • Communicates with the controller
    • Template Engines
  • Controller
    • Receives input (From View, URL)
    • Processes requests (GET, POST, POT, DELETE)
    • Get Data from the model
    • Passes data to the view

How to insert data into database via model and controller :

Two basic way to pass data into the database

  1. With JQuery Key word INSERT (How to write INSERT in SQL)
  2. with JSON file, and passed by method.

for pass JSON file, need to create a new model, and to have method to pass data.

load Json file to string, then pass to the model feature by feature.
Then add to the _db .

  • AppDbContext is the database ‘Model’.

Pass data to the final View page :

To pass data to the final view page we need 3 components controller, xModel and xViewModel.

In example I have :

  • Category.cs Model to hold data from DB.
  • CategoryModel.cs methods allow us access and opeate the model(Category.cs)
  • CategoryViewModel.cs View Model don’t see the database, it’s handle the data pass from the CategoryModel.
