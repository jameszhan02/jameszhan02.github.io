Declarative Language Review (All XML related)

DOM (Document Object Model):

Basic DOM componemnt: Nodes, NodeList, Document, Element, Attr, NodeList, NodeNamedMap

DOM view xml file as tree structure (nodes)

  • DOM is acturally xml file itself but with object model in different language

  • DOM is defined as a set of interfaces.

Components:

  • Node ( The fundamental interface in the DOM ):
    • nodeNmae : is the tag name(or attribute name for attributes)
    • nodeValue: text within the node.
    • nodeType: is an enumeration different node types( element, attr, text & etc.)
    • parentNode: is the node to which a given node belongs(may be null)
    • hasChildNodes: to check if there any child element inside
    • textContent: is the aggregation(collection) of the text of all children.

Pic show all of interface inside the node.

  • NamedNodeMap(belong to node)
    • getNameItem(“start”): return “3:00 pm” in this example. (return node item with key value attr name)
    • setNamedItem(node): requires that an attribute node be passed, will do add / replace the existing attributes.
    • removeNamedItem(“start”): would remove the entire start attribute in this example.

  • NodeList

    • a type will hold that when a method return more than one node. (eg. Document.getElementsByTagName(name))

    • length is the count of the list

    • index start from 0 as a nomal list. invalid index will return null.

  • Element
    • stright forward represent an elemenrt (Inherits from node)
    • extensions to node focus on attribute opreation (element is the only type able to handle / support attributes)
    • First four method is straight forward allow get set remove and check if its empty
    • last three method in the pic use the Attr class.

  • Attr
    • represent a single attribute ( Inherits form node )
    • only contaons a name, value and which element its belong to
    • Attr only has a single child and is text node. Attr.childNodes is always of length 1 and containing a text ndoe.

  • Document
    • all of instances of node belong to a single document. Document Itself inherits from Node.
    • its unique with able to create node instances, have to use Document type to add a model
    • first Five methods in the Document type used to create new node types

XSLT:

XSLT is the further version of XSL (EXtensible Style Sheets)

XML is the language to representing data,
XSLT is used to format xml file data with tags. to parsed before displaying it to separate the formatting format from the actual data

Think about HTML and Css. XSLT is just like css used in html but in xml. its help data with the layout we wanted and with right output type (xml, html & txt )

XSL is a style sheet language for XML

XSLT is further enhanced by XPath

XSLT are xml doc.

XSL is mainly used for formatting

XSLT can be used for formatting, but was designed to be a general conversion tool, converting one XML document to another (i.e. “processing” it)

XPath is a query and processing language made for XML documents (sort of like SQL for databases)
